# 显示客户端 Toast 提示 (Show Client Toast)

在客户端屏幕右上角显示一个标准的系统通知（Toast）。

## 节点概览
- **分类**: 客户端 > 界面与提示
- **内部ID**：`mgmc:client_show_toast`
- ![alt text](images/image.png)

## 端口定义

### 输入 (Inputs)
| 端口名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| **输入** (exec_in) | 执行流 (Exec) | 触发该节点的执行。 |
| **标题** (title) | 字符串 (String) | Toast 的第一行标题。默认为 "Maingraph"。 |
| **消息** (message) | 字符串 (String) | Toast 的第二行详细消息。 |

### 输出 (Outputs)
| 端口名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| **输出** (exec_out) | 执行流 (Exec) | 节点执行完成后触发后续节点。 |

## 行为说明
1. **主要行为**：当节点被触发时，服务端会将标题和消息参数打包发送给触发蓝图的玩家客户端。客户端接收后，使用 Minecraft 原生的 `SystemToast` 机制在屏幕右上角弹出提示。
2. **环境限制**：此节点仅在客户端生效。如果是在服务端执行（如全局蓝图），它会自动寻找触发该蓝图的玩家并发送包。
3. **空值处理**：如果“标题”未连接或为空，将显示默认值 "Maingraph"；如果“消息”为空，则仅显示标题。
4. **多线程安全**：客户端在收到请求后会确保在 Minecraft 主线程中执行 UI 操作，避免线程冲突。
