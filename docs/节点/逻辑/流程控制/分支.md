# 分支 (Branch)

- **分类**: 逻辑 -> 流程控制
- **内部 ID**: `branch`

## 节点功能
根据输入的布尔条件，决定逻辑执行的走向。它是实现“如果...那么...否则...”逻辑的核心。

![分支-节点样子](https://free.picui.cn/free/2026/01/03/6958409112984.png)

## 入参
| 端口名称 | 内部 ID | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| 输入 | `exec` | 执行 (EXEC) | 触发该节点。 |
| 条件 | `condition` | 布尔值 (BOOLEAN) | 决定走向的条件。 |

## 出参
| 端口名称 | 内部 ID | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| 为真 | `true` | 执行 (EXEC) | 当条件为 True 时执行。 |
| 为假 | `false` | 执行 (EXEC) | 当条件为 False 时执行。 |

## 使用示例

### 场景：根据血量给予提示
1. **[[获取实体信息]]**: 获取玩家的 `生命值`。
2. **小于 (<)**: 判断 `生命值` 是否小于 `10`。（[[数值比较]]）
3. **分支**: 将 `小于` 的结果连入 `条件`。
4. **[[打印到聊天栏]]**: 
   - `为真` 分支：发送“警告：血量过低！”。
   - `为假` 分支：发送“状态良好”。
